# 小说生产短规则手册

## 使用方式

本手册是脱敏实验样本的长期总结，不是每章都要加载的提示词。日常生产只注入
`literary-micro-rules/v3` 的角色短版；需要解释某条规则、复盘失败或升级规则生命周期
时才读取本文件。

规则分四类：

- **可以写**：通常能增加人物特异性、现场感和关系压力。
- **慎写**：不是禁令，但很容易变成机械语或过度设计。
- **允许**：不能因为不整齐、不完整或不够“聪明”而误判为缺陷。
- **绝对禁止**：会破坏正文、人类感或正式证据链。

## Writer

### 可以写

- 人物在压力中主动选择，并立刻损失某个人、关系、记忆、物件、身体能力或职业信用。
- 错误判断、迟疑、嘴硬、隐瞒、反应迟半拍，以及选择后没有立即被解释清楚的余波。
- 会改变动作的专业细节：材料状态、工具限制、身体负荷、工序风险、价格和时间死线。
- 状态会变化的物件。灯、门、绳、工具、证物、手中物和人物位置必须有可追踪的前后状态。
- 不以剧情效率为唯一目的的关系动作，例如照顾、回避、敷衍、欠情、迁怒和不肯说出口的话。

### 慎写

- 精确数字。数字若不改变决定、限制、风险、后果或必要结构，通常只是技术装饰。
- 过于完美的证据链。所有痕迹都恰好指向唯一结论，会让人物像在执行推理模板。
- 连续以人物全名开句、连续同构短句、连续冒号解释和连续“发现一个新证据”。
- 职业术语和真实知识。读者应先看到人物怎么做、为什么危险，再看到术语。
- 强钩子、奇观和漂亮结论。它们不能代替私人代价、关系变化和行动后果。
- 一章内过多巧合回报：证据、身世、贵人、对手和下一任务同时出现，会显得过度高效。

### 允许

- 人物不知道真相、说错话、做出次优选择，甚至暂时误解正确证据。
- 句子长短不均、纯对白、克制叙述、留白和不立即结算主题。
- 配角有自己的账，不帮助主角证明聪明，也不按主角需要及时提供答案。
- 专业操作失败、体力不足、工具不趁手、证据不完整和行动成本高于预期。
- 正文在现场中发现比规划更好的细节，只要不漂移用户硬锚，并保留因果与代价。

### 绝对禁止

- 把 Scene Package、因果链、替代解释、审稿意见、规则清单或主题结论翻译进正文。
- 用好奇、观察、偶然卷入或事后补救冒充主角的主动选择。
- 改写用户硬锚中的时间方向、数量、物件位置、人物知识来源、核心冲突或章末钩子。
- 用通用的“失去一些东西”“付出代价”替代已规划的具体私人损失。
- 在正文中出现 Agent、Session、Generation、Guardian、状态、哈希、审稿或其他生产信息。

## Blind Reader

### 可以认可

- 去掉姓名和职业后，人物仍因独特关系、习惯、羞耻、欲望和选择而不可替换。
- 读者能重建身体、空间、限制和行动，同时记住至少三个承载关系或后果的画面。
- 下一章吸引力既来自未知事实，也来自人物将如何承担已经发生的关系变化。

### 慎判

- “专业”“精确”“很牛”“信息密度高”都可能只是技术表演。
- 画面能复述不等于动作连续；必须核对灯、门、手中物、证物和人物位置。
- 句子好看不等于有人味；检查人物是否可能犯错，选择是否留下私人残留。

### 允许

- 留白、误判、克制、难以言明的情绪和暂时不闭合的因果。
- 高压场景中的短句或纯对白，只要权力、身体或空间仍在变化。
- 读者暂时不懂世界规则，只要人物当下的欲望、限制和选择可理解。

### 绝对不能 Pass

- 主角没有主动选择，或选择没有可见代价。
- 物件、空间、身体动作或人物知识发生可验证矛盾。
- 正文像规划答卷、审稿答辩、职业证明或整齐问答记录。
- Patch finding 被直接改写成解释段，形成明显修补接缝。

## Chapter Editor

### 可以保留

- 正文偏离 Scene Package，但现场发现显著增强人物、因果或收束，并且没有漂移用户硬锚。
- 人物误判和信息缺口，只要知识来源、替代可能和后果成立。
- 不解释全部机制的结尾，只要不可逆动作与追读压力已经成立。

### 慎审

- 从一条证据直接跳到唯一地点、唯一幕后人或唯一结论的因果跨越。
- 搬运重量、距离、时间、工序和司法或职业程序的物理可行性。
- 过度完美的物证、连续编号、恰好匹配的痕迹和每一步都成功的专业操作。
- 重复同主语开句、解释性独词句和机械三连，它们可能让强情节仍显得程序化。

### 允许

- 规划不是正文判决书。Editor 可以认可更好的现场方案，但必须说明收益、损失和新增风险。
- MAY 可以保留；只有不修改就破坏逻辑、人物选择、可读性或核心钩子的问题才是 MUST。
- 每轮复审重新全文阅读，允许发现上轮没有出现的新问题，但必须一次列全当前 MUST。

### 绝对禁止通过

- 用户硬锚漂移、物理状态矛盾、知识越界、不可逆后果未发生或主动选择不成立。
- 具体私人代价被抽象成任何人物都能承受的通用损失。
- 正文变化后继续使用旧 Review、旧完成凭证或旧正文绑定。
- Lead 代写正文、代做盲审、代做编辑审，或用不同字符串冒充独立原生会话。

## Orchestrator

- 没有可用 `SessionBackend` 时，在创建书目录前停止；不得自动降级为手工 formal。
- Writer、Blind Reader、Chapter Editor 必须使用不同的真实 session 和底层实例。
- 会话完成凭证绑定角色、章节、当前 Generation、当前正文和具体产物，只由活动编排器写入。
- 正文变化必须新建 Generation、新开两份 Review，并让旧证据按绑定自然 stale。
- 只有开放 MUST 才触发一次集中 Patch；MAY 和 advisory 不改正文。
- `ready` 前由 Sequence 见证精确候选；Sequence effective、状态和 Git 任一失败都不显示完成。
- Agent 产出的正文不得记录为 human；未知遥测保持 null，不得用人工声明补成真实 Harness 事实。

## 规则升级

单篇样本只能增加解释或 experimental 规则，不能直接新增 blocking 文学门。至少跨作品、
题材或模型重复出现，并证明误报成本可接受后，才考虑升级。流程真实性、不可变证据和
角色隔离属于机器合同，不受文学偏好影响。
