# Agent Demo 对照证据：模型、智能体与回炉成本

> 记录日期：2026-07-17。
>
> 用户授权清理 `books/mingyun`、`books/shentong-90`、`books/夜班怪谈`
> 三个完整试验项目。本文件只保留来源边界、正文指纹、工作流状态、声音指标、
> 优缺点、运行成本观察与可迁移结论，不保留全文。
>
> 三篇使用了不同故事、模型、Agent harness 和实际完成深度，因此是
> **工作流诊断样本**，不是严格控制变量的模型排行榜。模型评分、门禁通过和
> `ready` 均不代表作者批准、文学价值认证或发布许可。

## 样本一：《命运摆渡人》

**有效品质：**

- 城市日常物件密度高，钱、时间、通勤、账目和具体动作能稳定托住现实感。
- 主角通过计数、测量和核对形成了可辨认的人格习惯，不完全依赖履历说明。
- 6647 CJK 在约 3 分钟内完成，说明该组合的吞吐和任务收束很强。

**暴露的问题：**

- 对白占比仅 0.5%，人物关系主要靠内心解释，场景容易成为单人认知流水线。
- 反复解释“他在想什么、这意味着什么”，主题结算早于行动后果。
- 规划中的关键分支、因果责任与正文的未来片段展示没有形成足够强的可逆/不可逆链。
- 正文把 2026-07-17 写成星期四、把 2026-07-20 写成星期日；实际分别是星期五和
  星期一。旧场景包没有独立的日历/时间算术反证项。
- Voice Bible 禁止“他意识到/他终于明白”式解释导语，但 lint 只覆盖后者，
  暴露了模板与规则漂移。

**工作流诊断：**

正文已经存在，chapter-state 仍停在 `planned`，且 generation 未记录、review 为空。
旧 `project-status` 能显示这些字段，却没有把“有正文但仍 planned”聚合成明确的
完整性警报。

## 样本二：《九零神瞳》

**有效品质：**

- 三篇中物质触感最扎实：雨、水泥、旧物、硬币、电话机等细节能改变动作，而不只是装饰。
- 情绪较克制，较少直接宣布人物感受；比喻密度和对白比例也比 M2.7 更均衡。
- MiniMax-M3 比 M2.7 花了更长思考时间，正文在触觉、环境和段落控制上确有更细的处理。

**暴露的问题：**

- 时间线写“雨在晚上十一点停”，后文又落到晚上九点，属于章内硬冲突。
- 公用电话先响，随后主角又投币拨号；动作机制没有在规划预检或同源审核中被及时拦住。
- 带红印的硬币在正文中被先指代、后介绍，信息引入顺序倒置。
- 场景本应在关键决定后停止，正文仍继续解释和收束，削弱章末压力。
- 规划本身已经带入电话机制矛盾，六个同源角色围绕同一规划审稿，容易把“计划写了”
  误当成“正文成立”。

**工作流诊断：**

项目缺少 `planning/chapter-state/ch01.md`。六份 review 的 Markdown verdict 带粗体和
附加说明，当前解析结果不是合法精确 verdict；所有 review 也因旧式
`source_fingerprint`/generation 引用而 stale。旧状态查询只遍历 state 文件，因此
出现“正文和审稿都在，但 chapters 为空”的盲区。

## 样本三：《夜班怪谈》

**有效品质：**

- 三篇中商业类型结构最清晰：规则逐步出现、危险升级、章末换题明确。
- 对白占比最高，人物通过问答、试探和规则执行推进，读者更容易重建场景。
- 有限认知、动作后果和线索回收比另外两篇完整；最终版本通过了现有正式门禁。
- 比喻密度最低，文本较少依赖装饰性形容。

**暴露的问题：**

- 规则事件容易按清单顺序依次触发，结构可读但机械感偏强。
- 配角主要承担发规则、验证规则和制造下一事件的功能，独立欲望仍薄。
- “接下钥匙”的不可逆性不足，理论上仍可归还或拒绝后续值班。
- 四份 generation、三轮左右回炉和约 50 分钟耗时，说明质量链缺少收敛预算。
- 六个角色与生成完全同 provider/model；角色分工增加覆盖面，但不增加独立证据。
- 单章产生约 38 条 candidate 和约 38 条 Canon 记录，连续性记忆过度原子化，
  后续上下文可能被低价值事实淹没。
- 状态证据大量使用 `-`，且表内状态顺序不稳定，审计可读性不足。

**工作流诊断：**

这是三篇中唯一进入 `ready` 的项目，说明完整工作流确实提高了因果、规则和连续性
覆盖；同时它也最明显地展示了过度约束：审稿链没有跨来源独立性要求，返工没有停止
条件，记忆没有显著性预算，最终容易得到“很完整、很机械、很昂贵”的章节。

## 对工作流约束效果的判断

**已经有效约束：**

1. 正式章体量、基础标点和表面格式。
2. 场景目标、阻力、代价、有限认知和因果归属的显式规划。
3. 正文、规划、模式与审稿的哈希绑定；材料变化后 review 自动 stale。
4. 以身体、物件、动作和关系变化承载情绪，减少纯抽象总结。
5. exploration/formal 分流，以及 `ready` 不等于作者批准的边界。

**尚未有效约束：**

1. 规划本身的日历、常识、动作机制和知识来源错误。
2. 审稿者先看到规划后被意图锚定，把计划存在误判为正文兑现。
3. 同一模型换六个角色造成的“覆盖面”与“独立性”混淆。
4. 正文存在但状态缺失、状态停在 planned、verdict 被 Markdown 包裹等漂移。
5. 多轮审核 finding 重复、局部问题不断触发全文回炉，缺少固定预算和人工停止线。
6. generation 缺少耗时、token、暂停、交互、父 revision 和轮次元数据。
7. Canon/candidate 没有显著性与单章体量警报。

## 固化为下一版工作流的规则

1. 场景包新增“规划反证与常识检查”，逐项回答日历/时间、物理动作、知识来源、
   不可逆性和场景停止点。
2. causal-editor 与 chapter-editor 先只凭正文重建事件，再打开规划做差异审计。
3. `ready` 与 `benchmark_eligible` 分离；同源评审可完成本地生产链，但不能冒充独立基准。
4. `project-status` 合并发现正文与 state，输出 `workflow_integrity`、明确 blockers/warnings。
5. 正式模式默认自动执行一次完整审稿批次，不在“是否开始审核”处暂停。
6. 审核采用“一批发现 → 去重裁决 → 一次合并 patch → 一次全文终审”。
7. 初稿加两次后续 generation 即达到人工边界；再出现新 MUST 时进入
   `human_decision_required`，不自动进行第三次回炉。
8. generation evidence 可选记录耗时、token、暂停、交互、轮次、父 generation、
   阶段和来源置信度；未知值保持未知。
9. 记忆记录增加 salience，单章候选或 Canon 过量时给出 advisory，不自动删除事实。
10. “他/她意识到”与“他/她终于明白”共用一份 lint 模式来源，避免模板和实现漂移。
