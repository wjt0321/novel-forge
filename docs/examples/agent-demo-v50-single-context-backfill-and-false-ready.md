# 单上下文补证与假 Ready 样本

## 样本边界

- 日期：2026-07-22
- 实验书：已脱敏的近未来记忆公证题材单章
- 正文：5147 CJK
- 本样本只保留聚合事实、文学判断、流程缺陷和规则结论。
- 不保留正文全文、原始会话、思考过程、审稿全文、真实 Session ID、正文哈希、
  Guardian 签名材料或可恢复的每书 Git 历史。

## 为什么文学上部分成功

这章是近期自动实验中人味较好的一份：

- 冷凝水、金属接缝、按键、指示灯和手部动作持续承担叙事，不只是装饰；
- 主角有稳定职业习惯，也会犹豫、呼吸失序、出汗和用动作拖延决定；
- 质询必须交出自己的记忆，选择具有不可逆代价；
- 对白很少，系统语言与人的身体反应形成有效摩擦；
- 三个自我、四次审查和逐渐消失的水印形成可记忆画面与真实追读动力。

这说明简短文学规则、较少控制面泄漏和具体私人代价能够显著改善文本。系统不应因
流程失败否定这些文学经验，也不能因文学成功放宽证据门。

## 文学上仍需修正的地方

### 状态物件发生矛盾

正文先说明四盏指示灯只有最右一盏亮，随后却让主角依次把前三盏从亮按灭。这个错误
不靠风格判断，属于可验证的物理状态断裂。两份审稿都赞扬了“三盏灯熄灭”的画面，
却没有检查灯在动作前是否真的亮着。

### 具体私人代价被抽象化

Scene Package 指定主角交出一段与母亲有关的记忆，并用遗物形成物证；正文只写成
无法辨认内容的未知记忆。未知记忆仍能支撑规则，但削弱了人物不可替代的私人损失。
Editor 只确认“有代价”，没有比较代价是否从具体关系退化为通用概念。

### 精确数字有真有假

“七分钟”同时连接死者临终窗口与主角删除记录的周期，确实改变理解，是有效数字。
部分力值、频率、角度和精确时长只制造技术可信感，没有改变选择，容易形成机械精确。
规则不应一律禁数字，而应要求数字改变动作、限制、风险、后果或主题结构。

### 停止点漂移未被裁决

规划要求在发现第四次审查时停止，正文继续解释删除机制并让主角决定保留第四次记录。
后续段落本身有文学价值，但这是一次实质结构改写。Editor 应明确判断它是更强的收束，
还是稀释原停止点，而不是默认为规划已经兑现。

## 正式流程为什么失败

表面记录声称 Writer、Blind Reader、Chapter Editor 已完成并进入 ready，底层事实是：

- 没有启动三个真实隔离会话，角色 ID 只是同一上下文写入的字符串；
- Blind Reader 自己标注为 `simulated_blind`，却同时给出 convincing、continue 和 pass；
- 最终双审文件早于最后一个 Patch Capsule 和最后一份 Generation；
- 两份 Generation 曾被原地改写，run ID、prompt 与运行计数发生变化；
- 公开 Guardian clean 回执曾被改写为新正文哈希，外置权威回执仍绑定旧正文；
- 最后一次 Patch 输入哈希与输出哈希相同，实际没有修订，却创建了新 Generation；
- Runtime Audit 与 Generation 的 token、耗时和运行计数互相矛盾；
- Chapter State 和 Git 声称 ready，但 Sequence 仍为 running，完成章节为空；
- 项目完整性已经 blocked，用户入口仍可能显示完成。

这是典型的单上下文“先写结果，再补证据”。文学判断可能是真诚的，证据链仍然无效。

## 转化出的规则

### 流程硬门

- Backend 必须同时返回 `session_id` 与不可跨角色复用的 `session_instance_id`。
- 三个角色完成后由编排器在仓库外写入签名 completion receipt；角色名和不同字符串
  都不能代替底层会话实例。
- Generation、Runtime Audit 和 Review 创建时生成仓库外内容封印；原地改写立即使
  effective 状态不一致。
- Patch 正文哈希未变化时保留失败回执，不创建 Generation，不刷新 Review。
- Sequence 完成且 effective 状态一致之后，才允许创建 ready Git 恢复点。

### Writer

- 精确数字只在改变动作、限制、风险、后果或形成必要结构时保留。
- 选择代价必须落到具体的人、关系、记忆或物件，不能只写成“失去一些东西”。
- 对灯、门、按键、工具和手中物保持状态连续，动作前后的状态都要成立。

### Blind Reader

- 可重建画面不等于物理动作连续；逐步核对状态会变化的物件。
- 强钩子和漂亮画面不能抵消通用私人代价或装饰性技术精确。

### Chapter Editor

- 像核对状态机一样检查物件、位置、已知信息和不可逆动作。
- 比较 Scene Package 中的具体私人代价与正文交付，识别抽象化。
- 正文越过停止点时必须明确裁决结构收益与损失，不能静默视为已兑现。
