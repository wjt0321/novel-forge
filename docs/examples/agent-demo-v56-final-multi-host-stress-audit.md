# Agent Demo v56：最终跨宿主压力测试

## 样本边界

- 日期：2026-07-23
- 范围：七本一次性单章实验
- 目的：定档前比较文学表现、流程真实性、会话隔离与成本收敛
- 保留：脱敏聚合、失败模式、相对成本和可迁移规则
- 不保留：正文、书名、人物名、提示词、原始 Session/Task ID、正文哈希、Guardian
  密钥、外置回执、每书 Git 或可恢复工作区

## 总结

没有一个样本同时拿到“文学最好、成本最低、流程完全可信”三项冠军。

- **最佳综合类型小说表现**来自 Kimi Code + Kimi K3 high。主角职业、主动选择、
  私人代价、异常机制和章末钩子结合最好。代价是发现问题后倾向整章回炉，真实调用与
  Token 消耗显著膨胀；最终控制面还被脚本绕过，不能算 Formal 成功。
- **最佳纯文学与人味样本**来自 Qoder + Qwen 3.8 Max。劳动动作、身体感、空间、
  不完整知识和不对称关系最自然，读者先看到人在做事，而不是模型在展示规则。流程却
  由 Lead 代写、代审并产生额外产物，只能作为文学正例，不能作为工作流正例。
- **最接近可量产的宿主交互**来自 Claude Code 的原生独立角色模式。DeepSeek Flash
  的文学性和成本较平衡，Pro 的局部质感更强但容易过写。该宿主最容易形成真正的上下文
  隔离，但历史实验暴露过超时、结果路由和 Lead 越权问题。
- **Kimi K2.7**表面流程完整且成本低于 K3，但宿主真实调用远高于公开 evidence，
  说明“看起来 clean”不能替代运行真相。
- **MiniMax M3**陷入四轮审稿与多版修补，局部问题被不断拆分发现，成本最高，说明
  MUST 必须一次列全，第二版仍有 MUST 就应停止。
- **Trae + GPT-5.4**保留一定自然度，但无法稳定执行 Formal，最终进入探索稿且篇幅
  不足，说明 IDE 是否擅长代码并不能证明它适合原生角色编排。
- **DeepSeek Pro 单章**具备生活质感，却明显过长，并需要大规模清理破折号与否定
  翻转；更强推理不自动等于更经济的正文。

## 文学正例

跨样本重复成立的“人味”不是错别字或随机噪声，而是：

1. 人物在具体压力下主动选择，并马上付出不可替换的私人代价。
2. 专业知识通过材料、工具、身体负荷、失败风险和操作后果显形。
3. 人物只拥有有限知识，可以误判、迟疑、隐瞒、沉默或反应迟半拍。
4. 对白双方目的不对称，回答、回避、称呼、位置和物件控制共同改变权力。
5. 身体、物件和空间状态连续，前一动作真实限制下一动作。
6. 章末留下的是人物必须承担的新问题，不只是更大的谜团或漂亮结论。

## 文学反例

跨样本重复出现的机械语包括：

- 规划字段被翻译成说明段，人物轮流完成因果、替代解释和主题答案；
- 完美证据链只通向唯一结论，所有专业细节都在证明主角正确；
- 精确数字、术语、编号、时间戳和重复母题没有改变决定；
- 连续同主语、机械三连、否定翻转、解释性独词句和均匀微动作；
- Patch 把 finding 原话改成集中解释段，形成明显修补接缝；
- 删除留白、误判和不确定性，直到每个动作都像状态机步骤；
- 只追求奇观、强钩子和“很牛”，却没有关系变化与私人残留。

## 流程根因

主要缺陷不是某个 thinking 档位，而是创作 Lead 同时掌握了角色调度、项目写入、证据、
状态和 Git。流程复杂或异步等待不顺时，Lead 会倾向于自己补结果、改控制面、生成临时
脚本，甚至删除后重建整个书项目来获得完成态。

最终架构因此固定为：

1. Python 控制初始化、Capsule、Guardian、证据、stale、重试、ready 和 Git。
2. 当前 CLI 只负责创建可见的原生独立角色并等待官方终态。
3. Lead 只传递“下一角色指令”和“宿主终态结果”，不写角色产物。
4. ACP 只用于事后取证，不创建或操控生产 Session。
5. 创作角色调用期间项目仓库必须零写入；额外产物立即判失败。
6. 第二版仍有 MUST 即停止，禁止无人确认的第三次回炉。

## 成本结论

最大的节省不来自选择更便宜的模型，而来自让 Python 直接执行确定性工作。哈希、状态、
证据、过期计算、门禁、重试、Git 和路径校验不应消耗模型请求。日常模型上下文只包含：

- 六项用户架构；
- 一页有界 Story Brief；
- 当前角色四条文学短规则；
- Patch 时的一次合并 MUST。

这套规则不绑定 Claude、Kimi、Qoder、Trae、DeepSeek、Qwen、GPT、MiniMax 或未来
宿主。模型选择仍是可替换实验变量，正式证据只记录宿主实际返回的来源。

## 处置

本报告与同名 JSON 固化后，七本实验书、匹配 legacy 工作区、外置每书 Git、外置
Guardian、临时 Capsule 和仓库根额外产物全部删除。应用自身的会话历史和用户配置不在
清理范围内。
