## 审稿隔离失败

- 21/30 份审稿引用了未来章节。
- ch01-ch04 的 blind-reader 都提到 ch05，虽然元数据写
  `context_scope=prose_only`。
- ch01 审稿在 ACP 序号约 101-106 时写入；ch05 正文约序号 356 才开始。
- 没有发现隔离 reviewer session；30 份 review 均由父会话直接连续写入。

因此 `context_scope` 是模型自述，不是执行事实。角色名和 Markdown 字段不能证明隔离。

## 结论

机械状态全部进入 ready，但审计可信度、盲读隔离、文学表面和成本效率均失败。

该样本支持 v3.8 的四项改造：

1. 每章独立会话与 cached-input 预算。
2. 一次完整 Write + 最多一次集中 Edit。
3. 默认两角色：prose-only blind-reader + 综合 chapter-editor。
4. blocking workflow-meta-leak 与跨章句长/复读 advisory。

样本不证明 MiniMax M3 必然低质；它证明在 Claude Code 单一增长会话、high thinking 和
v3.7 重型流程组合下，模型把大量 token 用于缓存上下文、资产生成和自我证明。
