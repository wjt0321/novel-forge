## 文学判断

### 可保留的方向

- 开场异常、职业动作和章末钩子具备清楚的类型小说功能。
- 局部场景有可视物件和身体动作，不是纯设定摘要。
- 谜面可以理解，读者能知道人物正在处理什么危险。

### 失败表现

- Writer 把规划中的替代解释、因果检查和专业判断逐项搬进人物思考，人物像在
  展示推理报告，而不是在压力下生活。
- 信息释放过度完整，几乎每个疑点都被及时命名、分类和解释，缺少误读、遗漏、
  自欺和延迟反应。
- 对白承担问答式信息传递，轮次整齐；后续为躲避固定行数规则而机械插入动作，
  没有真正改变权力关系或潜台词。
- 人物现实关系和私人损失不足，主角换成另一个同职业角色后，大部分情节仍成立。
- Patch 直接解释审稿指出的问题，形成明显修补接缝，没有把动机分散回动作、停顿、
  关系和物件后果。
- Blind Reader 把“谜题可重建、钩子存在”误当成充分的人味与追读证明，没有识别
  控制面泄漏和解释性补丁。

## 根因

旧 Writer handoff 把完整 Scene Package 交给正文会话，其中包含编辑控制面的决策问题、
替代解释、可证伪假设、因果归属和专业判断审计。模型为了完成任务，会自然地在正文中
逐项证明这些检查，从而产生“看起来很厉害，但读完不知道人物活过什么”的空洞感。

Blind Reader 与 Chapter Editor 虽然拥有独立会话，但角色任务编译过短，没有明确识别
控制面泄漏、人物可替换性、整齐问答、解释性修补接缝，也没有要求 Chapter Editor 每轮
重新完成全部文学维度。模板中的固定对白行数规则进一步诱导机械动作插入。

实际宿主执行还绕过了正式 Orchestrator 的 Patch 会话轮换，因此初稿和 Patch 使用同一
Writer 上下文。模型看到自己的旧推理和修订目标后，更容易为原稿辩护，而不是重新体验
场景。

## 改造结论

- 完整 Scene Package 保留为编辑控制面；Writer 只接收经过过滤的 Story Brief。
- Writer 不再看到决策审计、替代解释、可证伪假设、因果归属和专业判断审计。
- 正文任务明确允许人物误判、遗漏、自欺和延迟反应，禁止逐条证明规划。
- 对白按身体位置、权力变化和回应关系判断，不再使用固定行数配额。
- Blind Reader 增加控制面泄漏、整齐问答、职业证明和修补接缝检查。
- Chapter Editor 每轮重新检查因果、能动性、对白、肌理和连续性，不得只核对旧 finding。
- Patch 指令绑定位置、原文证据、读者效果和修订意图，并禁止把 finding 改写成解释段。
- 初稿、每次 Patch、每位审稿角色都必须使用新的真实会话。
- 自动成本仍保持“一次初稿、最多一次自动集中 Patch、每轮两次审稿”；第三版必须由
  用户明确选择重新生成后才授权。
- 上述协议只定义能力和上下文边界，不绑定 Claude、Codex、DeepSeek 或任何特定模型。
