# GPT-5.4 / MiniMax / DeepSeek / GLM-5.2 正文完成度对比

## 实验边界

本轮没有可用 session export，只根据四个 `books/` 项目的正文、规划材料、门禁、
审稿、证据与序列文件评估。仓库保留同名 JSON 中的逐章 CJK、文件哈希、流程状态、
重复统计和结论；不保留正文、提示词、推理过程或工具内容。

四个样本分别为：

| 项目 | 模型 | 章节 | 正式篇幅 |
|---|---|---:|---|
| `weather-bureau-49` | GPT-5.4 | 3 | 全部不足 5000 CJK |
| `dorm-601-rules` | MiniMax | 2 | 全部达到 |
| `abyss-observer` | DeepSeek | 3 | 全部达到 |
| `rebirth-1979` | GLM-5.2 | 3 | 全部达到 |

## 文学结论

### GPT-5.4：最像人类写的类型小说初稿

《四十九号气象站》最懂得让章节同时完成局部事件和全书推进。配角拥有独立判断，
物件承担行动与社会环境，读者首先感受到的是故事，而不是写作规则。

它的主要失败是生产纪律：三章依次约 4142、3473、2934 CJK，篇幅持续缩水；没有
generation、runtime audit、正式审稿或 sequence。正文最好，但没有完成 formal
生产闭环。

### MiniMax：最服从流程，也最像状态机

《601 寝室守则》满足正式篇幅，规则、镜面和真名机制清楚，两章文件状态都达到
`ready`。但正文分别有 341 和 348 个非空叙事段，第二章反复使用时间戳、闭眼、
数呼吸、确认状态。

Voice Bible 把句长、对白比例和段内句数写成精确目标后，模型开始主动刷指标。
因此统计指标不能再作为 Writer 的生成目标，只能由 Editor 用作事后漂移诊断。

流程文件也没有完全闭合：runtime budget 已超限，第二章已经 ready，但 sequence
仍停在 running，completed chapters 只有第一章。

### DeepSeek：概念与氛围强，模型惯性压倒规则

《深渊观测者》完成了发现、回应、污染的三章递进，但三章 blocking lint 约为
74、94、104；破折号数量为 57、81、90，逐章恶化。“每分钟四次”累计出现约
34 次，母题从记忆点变成了反复证明。

其 evidence Markdown 缺少权威标记，`project-status` 无法解析；三个 blind review
均为 `needs_revision`，sequence 却声明 complete。完成状态与证据真相发生冲突，
说明序列状态不能信任 Agent 自报，必须由 ready、generation 与 session 绑定反推。

### GLM-5.2：局部质感好，Voice Bible 表层过拟合

《重生1979》三章均达到 5000 CJK，年代物件和物质约束丰富。但三章累计出现约
61 次“他想”、44 次“煤球”，并反复使用“扣子摁了两下”“他不去想”等动作。

这些重复与 Voice Bible 中的具体示例高度对应。模型把“用动作和物件承载情绪”
理解成了固定动作与固定物件。第二章末明确决定次日不去洼地大队，第三章开头却已经
在车站买票，中间没有新的触发事件；同时 ch02/ch03 正文已经存在，状态仍为
`planned`。

## 对 Harness 的诊断

四个样本对应四种不同失败：

1. GPT-5.4 没有充分受流程约束，但保住了文学自然度。
2. MiniMax 过度服从可量化目标，把正文写成指标执行结果。
3. DeepSeek 让自身修辞惯性越过硬禁令，并留下矛盾的完成状态。
4. GLM-5.2 服从正面示例的表层形式，把声音锚复制成了口头禅。

因此 v4.1 的原则是：

> 流程要硬，语言要松；事实要证明，文学不能刷分。

## v4.1 决策

- 推理分层：规划和疑难因果核验用 high；默认正文用 standard/medium；默认审稿
  用 standard/medium；max 只处理被明确命名的困难问题，不用于整章自由生成。
- Writer 不接收句长、段落、对白占比等数字目标。数字只用于审稿诊断，而且不得
  被解释为文学及格线。
- Voice exemplar 只传递叙事距离、信息释放和节奏功能；禁止复用其具体名词、
  标志动作、章末物件和句法骨架。
- 新增章内句首/完整句/短语饱和预警，并继续保持 advisory，不由机器认证文学价值。
- 上一章末明确决定在下一章被推翻时，必须在章际交接中写明触发事件，并真实落入
  当前正文。
- sequence 的 complete 必须与全部章节 ready、generation/run_id/session 绑定一致；
  evidence 无法解析或审稿未通过时不能报告健康完成。
- 中文正文中的 ASCII 逗号和直引号进入 surface advisory，不自动改写。

## 样本处置

原始四个 `books/` 项目仅为试错 demo。在本报告与同名 JSON 固化后删除；后续模型
比较只引用脱敏统计、哈希和结论。
