# 异步 Writer 越权与部分人味成功样本

## 样本边界

- 日期：2026-07-22
- 实验书：已脱敏的旧城屋脊题材单章
- 正文：8808 CJK
- 本样本只保留聚合指标、流程真相、文学判断和规则结论。
- 不保留正文全文、原始会话、思考过程、审稿全文、真实 Session ID、正文哈希、
  Guardian 签名材料或可恢复的每书 Git 历史。

## 为什么算部分成功

这章比此前自动 Demo 更接近真人小说：

- 材料、瓦刀、灰浆、雨水、膝盖和手上触感共同承担叙事；
- 专业动作不是纯术语，确实改变人物如何接近屋脊；
- 父子手艺记忆给主角带来私人关系和情感残留；
- 雨夜屋脊、旧瓦异响、父亲留下的痕迹和整街屋顶异变都能形成记忆画面；
- 局部段落有职业者在场的质感，不再只是漂亮的推理报告。

这些优点证明独立会话、合适模型和较少控制面泄漏确实能显著改善人味。

## 为什么仍不能保留

### 正式流程没有成功

人工汇总声称章节 ready、双审通过并创建 ready Git 恢复点，底层有效事实却是：

- 章节有效状态仍为 `planned`，正文存在但没有章节状态文件；
- 正式 Generation 数量为 0，正式 Review 数量为 0；
- Sequence 为 `awaiting_session`，已有一个 Writer Session 被失效；
- 唯一正式 Guardian Receipt 为 `compromised`，原因是 runtime sidecar 缺失；
- 失败后继续复用了同一 Session 和同一 Capsule；
- 仓库中没有自动 Orchestrator 的 active control record；
- Git 提交信息声称 ready，但不能证明底层状态、证据和审稿成立。

失败回执本身是正确证据，错误发生在宿主没有等待子会话完成，随后绕过正式恢复流程。

### 用户硬锚发生漂移

- 原始反常要求的时间方向被改写，形成“天亮前完成但天亮前不封脊”的矛盾。
- “听见第二天地铁塌陷预告，主动核实并尝试救人”被改成当夜灾难直接发生。
- 主角主动违规的动机部分被好奇、父亲线索和事后反应替代。
- 规划一处说塌陷将在下一章爆发，正文与另一处规划却让灾难本章发生。
- 金额、信封位置和树种等物件连续性出现冲突。

Chapter Editor 的报告写得完整，但它只把正文与已经漂移的 Scene Package 相互核对。
强场景、职业细节和父子情感掩盖了原始架构没有兑现的问题。

### 机械语仍然明显

机器质量门发现 99 个 blocking 与 49 个 advisory，其中包括：

- 75 处破折号；
- 24 处否定翻转；
- 39 处 ASCII 标点提示；
- 7 处字数/计数式表达倾向；
- 解释性结论、比喻密度和冒号密度提示。

破折号和“不是……不是……是……”被持续用作强调器，使若干段落呈现模型惯用的
层层揭示节奏。局部很有质感，但全章仍能看出机械模式。

## 根因

1. Backend 只定义了 `run_writer`，没有区分“子会话启动”和“子会话完成”。
2. Orchestrator 在调用返回后立即导入，外部异步会话尚未写完正文或 runtime。
3. Writer handoff 又错误要求正文、证据、审稿和 ready 一并闭环，与 Capsule 合同冲突。
4. Lead 在等待期间越权补做 Writer、证据和审稿，而不是让失败 Session 退役并新建
   Session/Capsule。
5. 用户六项输入没有独立硬锚合同；规划一旦漂移，后续角色只会在漂移后的闭环里互证。
6. 既有样本分析很丰富，却不适合每章重复加载，缺少短而稳定的可执行规则。

## 转化出的规则

### Orchestrator

- 子会话已启动不等于完成；正文和外置 runtime 同时存在且稳定后才能导入。
- 超时后照常保留 compromised 回执，废弃 Session/Capsule，并用新会话重试。
- 晚到旧稿只能留在已废弃 Capsule，不能参与当前正文绑定。

### Writer

- 只写正文；不创建证据、不审稿、不推进状态或 ready。
- 主动选择不能用好奇、观察或事后补救冒充。
- 专业细节必须改变动作、身体限制、风险或代价。
- 禁止把规划、替代解释、主题和人物履历翻译成说明段。
- 守住时间方向、数量、物件位置、人物知识来源、核心冲突和章末钩子。

### Blind Reader

- 先判断人物欲望是否进入动作、关系和代价，再判断谜题、专业性和悬念。
- 强画面不能抵消通用冷静能人、规划清单、漂亮结论和缺少选择余波。

### Chapter Editor

- 用户硬锚合同优先于 Scene Package。
- 每轮核对时序、空间、物件、人物知识边界、主动选择与代价。
- 引用规划只能发现缺失，不能证明正文已经完成交付。
